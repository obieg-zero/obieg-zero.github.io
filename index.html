obieg-zero

Browser-native document flow engine.
Zero backend. Zero config. Zero barrier.

Example

import { createFlow, templateNode } from '@obieg-zero/core'
import { ocrNode } from '@obieg-zero/ocr'
import { llmNode } from '@obieg-zero/llm'

const flow = createFlow()

flow.node('ocr', ocrNode({ language: 'pol' }))
flow.node('prompt', templateNode({
  template: 'Podsumuj: {{context}}'
}))
flow.node('llm', llmNode({
  modelUrl: '/models/Bielik-1.5B-v3.0-Instruct.Q4_K_M.gguf'
}))

// PDF → OCR → prompt → local LLM → answer
flow.set('file', pdfFile)
await flow.run('ocr', 'prompt', 'llm')
console.log(flow.get('answer'))

Philosophy

  • Zero backend — OPFS, IndexedDB, WASM. No server required.
  • Zero config — Sensible defaults. Plug in nodes and run.
  • Zero barrier — No accounts, no API keys, no setup. Works offline.
  • Zero bundle tax — 5 packages. Install only what you need.

How it works

  • Flow — orchestrator holding variables (shared state) and nodes (processing steps)
  • Nodes — read variables, do work, write variables. That's it.
  • Events — subscribe to variable changes, node lifecycle, progress updates
  • Dynamic imports — heavy deps (pdfjs, tesseract, wllama) loaded on first use
